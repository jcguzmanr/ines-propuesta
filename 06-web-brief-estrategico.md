## La Pregunta que Viene Antes del Brief

El cliente ha pedido un **landing de venta**.

Antes de diseñar, hay una pregunta estratégica que necesita respuesta:

> **¿El objetivo de la web es vender — o es que Inés sea percibida como lo que es?**

Estas dos cosas no son lo mismo. Y confundirlas tiene consecuencias sobre el posicionamiento, la inversión técnica y la experiencia del usuario.

---

## Disclaimer: Supuestos que Estamos Asumiendo

> ⚠️ **Nota importante para la presentación al cliente**
>
> Esta propuesta se desarrolla asumiendo ciertos aspectos del proceso de venta de Inés que **no han sido confirmados explícitamente**. Antes de avanzar con cualquier desarrollo web, es necesario validar los siguientes supuestos con el equipo de Inés:

### Supuestos sobre el modelo de venta actual

- Las ventas hoy ocurren principalmente vía Instagram (DM + a pedido)
- No existe un sistema de stock digitalizado / actualizado en tiempo real
- La producción es limitada y no siempre predecible en volumen
- El proceso de pago y despacho es gestionado manualmente o por plataformas de mensajería directa

### Supuestos sobre la intención de escalar

- Se quiere digitalizar el proceso de venta (catálogo online + compra)
- Hay intención de gestionar stock visible desde la web
- Existe (o se construirá) un proceso de actualización de inventario

**Si alguno de estos supuestos es incorrecto, el alcance técnico y el tipo de propuesta cambian significativamente.**

## Consideraciones Operativas: El Costo Oculto de Vender Online

Una landing de venta no es solo diseño. Detrás hay infraestructura operativa que alguien tiene que sostener:

### 1. Gestión de Stock
- ¿Quién actualiza el inventario cuando una pieza se vende?
- ¿Cómo se maneja una pieza que está "en proceso" vs "disponible"?
- ¿Qué pasa con las piezas únicas? ¿Desaparecen del catálogo al venderse?
- ¿Hay piezas "a pedido" que no están en stock pero pueden encargarse?

### 2. Fuente de Datos
- ¿Los datos de producto (fotos, descripción, precio, stock) viven en una hoja de cálculo, en un sistema externo (Shopify, WooCommerce, Notion), o se ingresan manualmente a la web?
- ¿Quién es responsable de mantener esos datos actualizados?
- ¿Con qué frecuencia cambia el catálogo?

### 3. Proceso de Compra
- ¿Se quiere compra directa con pago online (tarjeta, transferencia)?
- ¿O la web redirige a WhatsApp/DM para cerrar la venta manualmente?
- ¿Existe política de despacho, devoluciones, plazos?

### 4. Mantenimiento Post-lanzamiento
- ¿Quién actualiza la web después del lanzamiento?
- ¿Inés lo hace sola? ¿Hay alguien de soporte técnico?

> **Sin definir esto, una landing de venta puede volverse un problema operativo mayor que el que pretende resolver.**

## El Argumento Estratégico: Venta vs. Premium

### El brief del cliente dice "landing de venta"
### El manual de marca dice otra cosa

> *"No hay urgencia. No hay insistencia comercial. Las palabras acompañan al objeto. La pieza es siempre el centro."*
> — Brand Development INÉS 2026

Una **landing de venta** opera con lógica de conversión: CTAs directos, precio visible, urgencia implícita, catálogo navegable. Esa lógica funciona para volumen. **No funciona para joyería de autor high-end.**

Una **web de posicionamiento Premium** opera con lógica de deseo: el objeto primero, la historia después, la posibilidad de adquirir al final. Sin fricción comercial. Eso es coherente con lo que Inés construyó.

### La consecuencia directa

| Dimensión | Landing de Venta | Web Premium |
|-----------|-----------------|-------------|
| Objetivo | Conversión inmediata | Deseo + conversión diferida |
| CTA principal | "Comprar ahora" / "Agregar al carrito" | "Ver pieza" / "Solicitar" / "Contactar" |
| Precio | Visible y central | Visible pero secundario |
| Stock | Requiere gestión en tiempo real | Catálogo curado, sin urgencia |
| Mantenimiento | Alto (stock, pedidos, pagos) | Medio (colección, historia) |
| Riesgo de marca | Alto (puede percibirse como tienda masiva) | Bajo (refuerza posicionamiento) |
| Coherencia con brand manual | Baja | Alta |

### El punto central

> Si la web convierte agresivamente, **baja el precio percibido de la pieza.**
> Si la web eleva la percepción, **el precio se sostiene solo.**

Para Inés, el precio de una pieza no se justifica con un botón de compra. Se justifica con la historia, el material, el proceso. **La web tiene que hacer ese trabajo antes de vender.**

## Las Dos Propuestas

Se desarrollarán en documentos separados, pero aquí el marco:

### Propuesta A — Landing de Venta *(lo que el cliente pidió)*
Web orientada a conversión directa. Catálogo con precios, stock, flujo de compra. **Requiere infraestructura operativa definida antes de diseñar.** Riesgo: erosión del posicionamiento si no se ejecuta con cuidado extremo de la estética.

### Propuesta B — Web de Marca Premium *(lo que la marca necesita)*
Web como extensión de la identidad de Inés. El objeto es el protagonista. La compra es posible pero no urgente. Puede tener e-commerce o redirigir a contacto directo. **Coherente con el brand manual al 100%.** Menor complejidad operativa, mayor impacto sobre percepción.

## Preguntas que Hay que Responder Antes de Proponer

1. ¿El proceso de compra hoy es 100% manual (DM, WhatsApp)? ¿O ya existe algún sistema?
2. ¿Se quiere compra online con pago integrado, o basta con un catálogo que facilite el contacto?
3. ¿Hay alguien que pueda mantener el stock actualizado en la web?
4. ¿Cuántas piezas activas tiene el catálogo en este momento?
5. ¿Qué resultado concreto se espera de la web en los primeros 6 meses?
